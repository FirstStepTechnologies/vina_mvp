# Generated Media Files Strategy

## Overview

This project generates various media files (videos, images, audio) during development and testing. These files are **NOT committed to Git** to keep the repository clean and efficient.

## What's Excluded

The following file types are excluded from Git (see `.gitignore`):

- **Videos**: `*.mp4`
- **Audio**: `*.mp3`, `*.wav`
- **Images**: `*.png`, `*.jpg`, `*.jpeg`

## Why This Strategy?

### âœ… Benefits

1. **Smaller Repository**: Binary files don't bloat Git history
2. **Faster Operations**: Clones, pulls, and pushes are much faster
3. **No Merge Conflicts**: Binary files don't cause merge issues
4. **Easy Regeneration**: All files can be regenerated by running tests/demos
5. **Clean History**: Git diffs focus on code, not binary changes

### ðŸ“Š Impact

- **Before**: ~50+ MB of binary files in Git
- **After**: ~0 MB (only code and configs)
- **Regeneration Time**: ~3-5 minutes for full demo

## How to Regenerate Files

### Quick Test (30 seconds)
```bash
# Test individual components
python scripts/test_imagen_client.py
python scripts/test_tts_client.py
python scripts/test_slide_composer.py
python scripts/test_video_renderer.py
```

### Full Pipeline Test (3-5 minutes)
```bash
# Test complete end-to-end pipeline
python scripts/test_end_to_end_pipeline.py
```

### Complete Demo (3-5 minutes)
```bash
# Run the full hackathon demo
python scripts/demo_complete_pipeline.py
```

## For New Team Members

After cloning the repository:

1. **Install dependencies**:
   ```bash
   uv sync
   ```

2. **Set up environment**:
   ```bash
   cp .env.example .env
   # Add your API keys
   ```

3. **Install FFmpeg** (for video rendering):
   ```bash
   brew install ffmpeg  # macOS
   # or
   apt-get install ffmpeg  # Linux
   ```

4. **Run a test** to generate sample files:
   ```bash
   python scripts/demo_complete_pipeline.py
   ```

## Optional: Keep One Demo Video

If you want to keep a specific demo video in Git (e.g., for presentations), uncomment this line in `.gitignore`:

```gitignore
# !cache/demo_videos/demo_sample.mp4
```

Then commit that specific file:
```bash
git add -f cache/demo_videos/demo_sample.mp4
git commit -m "docs: Add demo video for presentation"
```
